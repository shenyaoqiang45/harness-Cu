# 伦敦铜走势预测系统：一般方案

## 一、目标

本系统不是预测某个精确价格点位，而是判断伦敦铜在未来一段时间内的多空力量强弱。

核心输出：

1. 未来 1 周走势：偏多 / 中性 / 偏空
2. 未来 1 月走势：偏多 / 中性 / 偏空
3. 主要支撑因素
4. 主要压制因素
5. 数据健康度
6. 判断失效条件

系统定位：

- 研究辅助系统
- 交易决策参考系统
- 风险预警系统

不做：

- 自动交易
- 精确点位预测
- 无来源数据判断
- 黑箱模型直接给结论

---

## 二、基本原理

伦敦铜价格由多股力量共同决定：

```text
铜价走势 =
中国需求
+ 全球库存与现货紧张度
+ 美元与实际利率
+ 全球制造业周期
+ 供应扰动
+ 风险偏好
+ 价格趋势
- 过热与拥挤惩罚
```

核心思想：

```text
将每股力量转成 -1 到 +1 的多空分数，
再按权重合成为总分。
```

最终输出不是“必涨”或“必跌”，而是：

```text
方向判断 + 置信度 + 原因 + 风险 + 失效条件
```

---

## 三、核心力量与权重

中期配置版本建议权重：

| 模块 | 权重 | 说明 |
|---|---:|---|
| 中国需求 | 30% | 铜的核心需求变量 |
| 库存与现货紧张度 | 25% | 实物供需状态 |
| 美元与实际利率 | 20% | 金融定价压力 |
| 全球制造业周期 | 10% | 外需与工业周期 |
| 供应扰动 | 10% | 矿端和冶炼端冲击 |
| 价格趋势 | 5% | 市场自身投票 |

短线交易版本可提高价格趋势、美元、风险偏好的权重。MVP 先采用中期配置版本。

## 四、MVP 核心指标

第一版不追求指标多，先保证可靠、可解释、可回测。

### 1. 价格趋势

指标：

- LME 铜价
- 20 日均线
- 60 日均线
- 120 日均线
- 20 日收益率
- 60 日收益率

量化规则：

```text
价格 > 20 日均线：+1
价格 > 60 日均线：+1
价格 > 120 日均线：+1
20 日收益率 > 0：+1
60 日收益率 > 0：+1
```

模块分数：

```text
trend_score = 多空得分平均值
```

### 2. 库存与现货紧张度

指标：

- LME 库存
- SHFE 库存
- COMEX 库存
- 全球显性库存 = LME + SHFE + COMEX
- 现货升贴水
- 期限结构

量化规则：

```text
全球库存 20 日下降：+1
全球库存 60 日下降：+1
库存处于三年低分位：+1
现货升水：+1
backwardation：+1
```

反向规则：

```text
全球库存 20 日上升：-1
全球库存 60 日上升：-1
库存处于三年高分位：-1
现货贴水：-1
contango：-1
```

### 3. 中国需求

指标：

- 官方制造业 PMI
- 新订单 PMI
- 社融
- M1
- 电网投资
- 地产竣工
- 铜材开工率

MVP 可先使用：

```text
PMI
新订单 PMI
社融 / M1
电网投资
```

量化规则：

```text
PMI > 50：+1
PMI 环比改善：+1
新订单 PMI > 50：+1
社融或 M1 改善：+1
电网投资同比为正：+1
```

负向同理。

月度指标需记录发布时间，不能把过期数据当成新信息。

### 4. 美元与实际利率

指标：

- 美元指数 DXY
- 美国 10 年实际利率
- 美国 10 年名义利率
- 通胀预期
- 降息概率

MVP 先使用：

```text
DXY
美国 10 年实际利率
```

量化规则：

```text
DXY 20 日变化 < 0：+1
DXY 60 日变化 < 0：+1
实际利率 20 日变化 < 0：+1
实际利率 60 日变化 < 0：+1
```

反向规则：

```text
DXY 上行：-1
实际利率上行：-1
```

### 5. 全球制造业周期

指标：

- 全球制造业 PMI
- 美国 ISM 制造业
- 美国 ISM 新订单
- 欧洲制造业 PMI
- 韩国出口

MVP 先使用：

```text
美国 ISM 制造业
全球制造业 PMI
韩国出口同比
```

量化规则：

```text
PMI > 50：+1
PMI 环比改善：+1
韩国出口同比改善：+1
```

### 6. 供应扰动

指标：

- TC/RC
- 智利铜产量
- 秘鲁铜产量
- 矿山罢工
- 冶炼厂减产

MVP 可以半人工录入：

```csv
date,event,score,confidence,source,note
2026-06-29,chile_mine_strike,+1,A,official_news,confirmed strike
```

规则：

```text
TC/RC 下降：+1
TC/RC 处于低分位：+1
重大矿山扰动：+1 或 +2
冶炼减产：+1
```

无可靠来源则不得加分。

## 五、总分模型

每个模块分数范围：

```text
-1 到 +1
```

总分：

```text
total_score =
0.30 * china_demand_score
+ 0.25 * inventory_score
+ 0.20 * macro_liquidity_score
+ 0.10 * global_cycle_score
+ 0.10 * supply_score
+ 0.05 * trend_score
```

判断规则：

| 总分 | 判断 |
|---:|---|
| >= +0.50 | 看多 |
| +0.20 到 +0.50 | 偏多 |
| -0.20 到 +0.20 | 中性 |
| -0.50 到 -0.20 | 偏空 |
| <= -0.50 | 看空 |

---

## 六、置信度

置信度不等于方向。

方向由总分决定，置信度由三部分决定：

```text
confidence =
abs(total_score)
* data_quality
* factor_consistency
```

### 1. 数据质量

```text
data_quality =
0.4 * source_score
+ 0.3 * freshness_score
+ 0.2 * cross_check_score
+ 0.1 * completeness_score
```

含义：

- source_score：来源可靠度
- freshness_score：数据新鲜度
- cross_check_score：多源交叉验证
- completeness_score：缺失程度

### 2. 因子一致性

```text
positive_count = 模块分数 > +0.2 的数量
negative_count = 模块分数 < -0.2 的数量
active_count = positive_count + negative_count

factor_consistency =
abs(positive_count - negative_count) / active_count
```

如果多空因子严重分裂，置信度下降。

## 七、数据可靠性原则

数据必须分层：

```text
raw_data
validated_data
clean_data
```

### raw_data

原始数据，只追加，不修改。

### validated_data

通过格式、单位、范围、日期校验的数据。

### clean_data

模型真正使用的数据。

每条数据必须带元数据：

```csv
date,indicator,value,unit,source,source_url,updated_at,frequency,confidence
```

必须记录：

- 来源
- 单位
- 更新时间
- 频率
- 可信等级
- 是否人工确认

核心规则：

```text
无来源，不入库
单位不明，不入库
异常波动，进入待复核
过期数据，降低权重
核心数据至少双源校验
```

---

## 八、异常数据处理

异常数据不得直接进入模型。

示例规则：

```text
LME 铜单日涨跌幅 > 8%：待复核
LME 库存单日变化 > 20%：待复核
DXY 单日变化 > 3%：待复核
PMI 超出 35-65：待复核
价格或库存为负：拒绝入库
```

数据状态：

```text
pending
confirmed
rejected
revised
```

只有 confirmed 数据可以进入 clean_data。

## 十一、第一版功能边界

MVP 必须实现：

1. 读取 CSV 数据
2. 校验字段、单位、来源、日期
3. 计算各模块分数
4. 计算总分
5. 计算置信度
6. 生成 Markdown 报告
7. 输出异常数据日志

MVP 暂不实现：

1. 自动联网抓数
2. 机器学习模型
3. Web 前端
4. 自动交易
5. 复杂数据库
6. 实时行情

## 十二、后续扩展

### 第二阶段：回测

增加：

```text
历史预测记录
未来 20 日收益率
命中率
最大回撤
分数分组表现
```

检验：

```text
总分越高，未来收益是否越高？
偏多信号是否有正期望？
偏空信号是否能规避回撤？
```

### 第三阶段：机器学习

在规则模型稳定后，再加入：

```text
逻辑回归
随机森林
XGBoost
时间序列交叉验证
特征重要性
```

禁止随机切分训练集和测试集。

采用 walk-forward：

```text
2015-2020 训练，2021 测试
2015-2021 训练，2022 测试
2015-2022 训练，2023 测试
```

### 第四阶段：前端面板

展示：

```text
当前判断
模块分数
历史总分
数据健康度
异常数据
历史命中率
```

## 十三、成功标准

MVP 成功标准：

```text
给定一份标准 CSV，
系统可以生成一份可读的伦敦铜走势判断报告。
```

最低可验收结果：

1. 数据异常能被识别
2. 每个模块有明确分数
3. 总分能生成方向判断
4. 置信度能反映数据质量和因子一致性
5. 报告能说明为什么偏多或偏空
6. 报告包含失效条件

---

## 十四、设计原则

1. 先数据纪律，后模型复杂度。
2. 先规则模型，后机器学习。
3. 先方向判断，后价格预测。
4. 所有结论必须可解释。
5. 所有数据必须可追溯。
6. 每次判断必须写失效条件。
7. 不因数据缺失而伪造确定性。
8. 不让新闻情绪直接替代核心数据。
9. 不把过期月度数据当成每日新信号。
10. 不在未经回测前提高模型权重。

---

## 十五、推荐第一步

新项目先实现以下闭环：

```text
CSV 输入
→ 数据校验
→ 指标计算
→ 模块打分
→ 总分判断
→ Markdown 报告
```

第一版只需要本地运行：

```bash
python -m copper_forecast.cli --input data/sample.csv --output reports/latest.md
```

只要这条链路跑通，就可以开始逐步增加真实数据源和回测。
